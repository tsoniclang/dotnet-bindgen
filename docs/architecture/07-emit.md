# Emit Phase

This document details the Emit phase, which generates TypeScript declaration files from the validated EmissionPlan.

## Overview

**Purpose:** Generate TypeScript `.d.ts` files from the EmissionPlan

**Input:** EmissionPlan (validated by PhaseGate)
**Output:** TypeScript declaration files on disk

**Key Files:**
- `Emit/FacadeEmitter.cs` - Public-facing facade files
- `Emit/InternalIndexEmitter.cs` - Internal declaration files
- `Emit/Printers/ClassPrinter.cs` - Type declaration printing
- `Emit/Printers/MethodPrinter.cs` - Method/property printing
- `Emit/Printers/TypeRefPrinter.cs` - Type reference printing
- `Emit/TypeNameResolver.cs` - Type name resolution

## Output Structure

For each namespace, tsbindgen generates:

```
output/
├── System/
│   ├── index.d.ts                    # Public facade
│   └── internal/
│       └── index.d.ts                # Full declarations
├── System.Collections.Generic/
│   ├── index.d.ts
│   └── internal/
│       └── index.d.ts
└── ... (one per namespace)
```

### Facade Files (`index.d.ts`)

Public-facing module that consumers import from:

```typescript
// Generated by tsbindgen - Architecture
// Namespace: System.Collections.Generic

import * as Internal from './internal/index.js';

// Cross-namespace type imports
import type { IEnumerable } from '../System.Collections/index.js';

// Re-export everything from internal
export * from './internal/index.js';

// Friendly aliases
export type List<T> = Internal.List_1<T>;
export type Dictionary<TKey, TValue> = Internal.Dictionary_2<TKey, TValue>;
```

### Internal Files (`internal/index.d.ts`)

Full type declarations with the `$instance` pattern:

```typescript
// Generated by tsbindgen - Architecture
// Namespace: System.Collections.Generic

import type { int, long } from '@tsonic/types';
import * as System_Internal from '../../System/internal/index.js';

// CLROf utility
export type CLROf<T> = ...;

// Type declaration
export interface List_1$instance<T> {
    readonly count: int;
    add(item: T): void;
    // ...
}

// Value export
export declare const List_1: {
    new <T>(): List_1<T>;
    // ...
};

// Views interface
export interface __List_1$views<T> {
    As_IEnumerable_1(): IEnumerable_1$instance<T>;
    // ...
}

// Type alias (intersection)
export type List_1<T> = List_1$instance<T> & __List_1$views<T>;
```

## FacadeEmitter

Generates public-facing `index.d.ts` files:

```
Emit(ctx, plan, outputDirectory)
    │
    └─► For each namespace:
        │
        ├─► Generate header
        │
        ├─► Import from internal
        │   └─► import * as Internal from './internal/index.js';
        │
        ├─► Cross-namespace type imports
        │   └─► import type { ... } from '../OtherNamespace/index.js';
        │
        ├─► Re-export from internal
        │   └─► export * from './internal/index.js';
        │
        ├─► Individual type exports
        │   └─► Value exports for classes/structs
        │   └─► Type aliases for interfaces/delegates
        │
        ├─► Friendly aliases
        │   └─► export type List<T> = Internal.List_1<T>;
        │
        └─► Delegate aliases (System only)
            └─► Action, Func with TS lambda compatibility
```

### Value vs Type-Only Exports

Different type kinds get different export treatment:

| Type Kind | Export Style | Reason |
|-----------|--------------|--------|
| Class | Value export | Construction (`new T()`) and static access |
| Struct | Value export | Construction and static access |
| StaticNamespace | Value export | Static method access (`Console.writeLine()`) |
| Enum | Value export | Enum member access (`ConsoleColor.red`) |
| Interface | Type alias | No runtime value |
| Delegate | Type alias | Function signature only |

### Delegate Aliases

In the `System` namespace, special `Action` and `Func` aliases are generated for TypeScript lambda compatibility:

```typescript
declare const __unspecified: unique symbol;
export type __ = typeof __unspecified;

export type Action<T1 = __, T2 = __, ...> =
  [T1] extends [__] ? ((() => void) | Internal.Action) :
  [T2] extends [__] ? ((arg1: T1) => void) | Internal.Action_1<T1>) :
  // ... etc
```

This enables both arrow functions and CLR delegates to be used interchangeably.

## InternalIndexEmitter

Generates full `internal/index.d.ts` declarations:

```
Emit(ctx, plan, outputDirectory)
    │
    └─► For each namespace:
        │
        ├─► Generate header
        │
        ├─► Branded primitive imports
        │   └─► import type { int, long, ... } from '@tsonic/types';
        │
        ├─► Support types (if needed)
        │   └─► import type { ptr, ref } from '@tsonic/types';
        │
        ├─► Cross-namespace imports
        │   └─► import * as System_Internal from '../../System/internal/index.js';
        │   └─► import type { ... } from '../../OtherNs/internal/index.js';
        │
        ├─► CLROf utility
        │
        └─► For each type (in order):
            │
            ├─► If CLR primitive:
            │   └─► export type Int32 = int;
            │
            ├─► If type has views:
            │   ├─► ClassPrinter.PrintInstance()
            │   ├─► ClassPrinter.PrintValueExport()
            │   ├─► EmitCompanionViewsInterface()
            │   ├─► EmitMergedInterfaceExtends()
            │   └─► EmitIntersectionTypeAlias()
            │
            └─► If type without views:
                ├─► ClassPrinter.Print()
                ├─► ClassPrinter.PrintValueExport()
                ├─► EmitMergedInterfaceExtends() (interfaces)
                └─► Emit type alias
```

## $instance Pattern

Types are emitted with a split between instance interface and value export:

```typescript
// Instance interface (shape of instances)
export interface List_1$instance<T> {
    readonly count: int;
    add(item: T): void;
}

// Value export (constructors and static methods)
export declare const List_1: {
    new <T>(): List_1<T>;
    new <T>(capacity: int): List_1<T>;
};

// Type alias (combines both)
export type List_1<T> = List_1$instance<T> & __List_1$views<T>;
```

**Why this pattern?**

1. **Separation of concerns** - Instance shape vs construction
2. **TypeScript hoisting** - Type aliases are hoisted, enabling forward references
3. **View composition** - Intersection types allow view accessors
4. **Static members** - Const declaration captures static side

## Views Pattern

When a type has explicit interface views (for non-structural members):

```typescript
// Main instance interface
export interface Decimal$instance {
    toString(): string;
    // ... ClassSurface members
}

// Companion views interface
export interface __Decimal$views {
    As_IConvertible(): __Decimal$As_IConvertible;
    // View accessor methods
}

// View interface (ViewOnly members)
interface __Decimal$As_IConvertible {
    toBoolean(provider: IFormatProvider): boolean;
    toByte(provider: IFormatProvider): byte;
    // ... ViewOnly members
}

// Final type alias
export type Decimal = Decimal$instance & __Decimal$views;
```

## ClassPrinter

Prints type declarations:

### Print() - Full Type Declaration

```
ClassPrinter.Print(type, resolver, ctx, ...)
    │
    ├─► TypeKind.Interface:
    │   └─► Print interface with $instance suffix
    │
    ├─► TypeKind.Class/Struct:
    │   └─► Print interface + const (via PrintInstance + PrintValueExport)
    │
    ├─► TypeKind.Enum:
    │   └─► Print const enum
    │
    ├─► TypeKind.Delegate:
    │   └─► Print type alias with call signature
    │
    └─► TypeKind.StaticNamespace:
        └─► Print abstract class (static methods only)
```

### PrintInstance() - Instance Interface

Prints the `$instance` interface for classes/structs:

```typescript
interface List_1$instance<T> {
    // Instance members (EmitScope.ClassSurface, !IsStatic)
    readonly count: int;
    add(item: T): void;
    // ...
}
```

### PrintValueExport() - Static Side

Prints the const declaration for constructors and static members:

```typescript
export declare const List_1: {
    // Constructors
    new <T>(): List_1<T>;
    new <T>(capacity: int): List_1<T>;

    // Static members
    readonly empty: List_1<unknown>;
};
```

## TypeRefPrinter

Prints type references:

```
TypeRefPrinter.Print(typeRef, resolver, ctx)
    │
    ├─► NamedTypeReference:
    │   └─► Resolve name via TypeNameResolver
    │   └─► Print type arguments with CLROf wrapping
    │
    ├─► GenericParameterReference:
    │   └─► Check against allowed names
    │   └─► Return parameter name
    │
    ├─► ArrayTypeReference:
    │   └─► Print as element[]
    │
    ├─► PointerTypeReference:
    │   └─► Print as ptr<element>
    │
    ├─► ByRefTypeReference:
    │   └─► Print as { value: ref<element> }
    │
    └─► PlaceholderTypeReference:
        └─► Print as "any" (error case)
```

### CLROf Wrapping

Primitive type arguments are wrapped with `CLROf<>` to satisfy CLR interface constraints:

```typescript
// Input: List<int> implementing IEnumerable<int>
// Output: IEnumerable_1<CLROf<int>>

// CLROf<T> utility:
export type CLROf<T> =
    T extends int ? Int32 :
    T extends long ? Int64 :
    T extends string ? String :
    // ... etc
    T;
```

This allows `List<int>` to be assignable to `IEnumerable<Int32>`.

## TypeNameResolver

Resolves type references to their TypeScript names:

```
Resolve(clrFullName, namespace)
    │
    ├─► Check primitive mapping
    │   └─► System.Int32 → int
    │   └─► System.String → string
    │
    ├─► Check local namespace
    │   └─► Return final name from Renamer
    │
    ├─► Check cross-namespace
    │   └─► Return qualified name from ImportPlan
    │
    └─► Check built-in
        └─► System.Object → unknown
        └─► System.Void → void
```

### Qualified Names

Cross-namespace references are qualified with namespace alias:

```typescript
// In System.Security.Principal namespace
// Referencing System.Security.Claims.ClaimsIdentity

// Value position (extends):
extends System_Security_Claims_Internal.ClaimsIdentity$instance

// Type position:
ClaimsIdentity  // via import type { ClaimsIdentity }
```

## Merged Interface Extends

For LINQ assignability, classes emit merged interface declarations:

```typescript
// Enables: List<int> assignable to IEnumerable<int>
export interface List_1$instance<T>
    extends IEnumerable_1$instance<CLROf<T>>,
            ICollection_1$instance<CLROf<T>>,
            IList_1$instance<CLROf<T>> {}
```

Uses `SafeToExtend` analysis to only extend safe interfaces (no TS2430/TS2320).

## CLROf Utility

Each namespace emits a `CLROf<T>` utility type:

```typescript
export type CLROf<T> =
    T extends int ? Int32 :
    T extends long ? Int64 :
    T extends float ? Single :
    T extends double ? Double :
    T extends decimal ? Decimal :
    T extends byte ? Byte :
    T extends sbyte ? SByte :
    T extends short ? Int16 :
    T extends ushort ? UInt16 :
    T extends uint ? UInt32 :
    T extends ulong ? UInt64 :
    T extends char ? Char :
    T extends boolean ? Boolean :
    T; // Identity fallback
```

**Purpose:** Maps ergonomic TS primitives to CLR types for generic constraints.

## Primitive Type Aliases

CLR primitive types are emitted as simple aliases:

```typescript
// System namespace
export type Int32 = int;
export type Int64 = long;
export type String = string;
export type Boolean = boolean;
// etc.
```

This enables bidirectional assignability between primitive aliases and branded types.

## Call Graph

```
Builder.EmitPhase()
    │
    ├─► FacadeEmitter.Emit()
    │   └─► For each namespace:
    │       └─► GenerateFacade()
    │           ├─► EmitImports()
    │           ├─► EmitReExports()
    │           ├─► EmitFacadeExport() per type
    │           └─► AppendDelegateAliases() (System)
    │
    └─► InternalIndexEmitter.Emit()
        └─► For each namespace:
            └─► GenerateNamespaceDeclaration()
                ├─► EmitBrandedPrimitiveImports()
                ├─► EmitCrossNamespaceImports()
                ├─► EmitCLROfUtility()
                │
                └─► For each type:
                    ├─► [primitive] → export type T = ts_primitive
                    │
                    ├─► [with views] →
                    │   ├─► ClassPrinter.PrintInstance()
                    │   ├─► ClassPrinter.PrintValueExport()
                    │   ├─► EmitCompanionViewsInterface()
                    │   ├─► EmitMergedInterfaceExtends()
                    │   └─► EmitIntersectionTypeAlias()
                    │
                    └─► [without views] →
                        ├─► ClassPrinter.Print()
                        ├─► ClassPrinter.PrintValueExport()
                        └─► EmitMergedInterfaceExtends()
```
