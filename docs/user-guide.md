## Installation

### Prerequisites

- .NET 10 SDK
- Node.js 18+ (for validation scripts)

### Building from Source

```bash
git clone https://github.com/tsoniclang/tsbindgen
cd tsbindgen
dotnet build src/tsbindgen/tsbindgen.csproj
```

### Verifying Installation

```bash
dotnet run --project src/tsbindgen/tsbindgen.csproj -- --help
```

## Basic Usage

### Generate BCL Declarations

Generate TypeScript declarations for the entire .NET Base Class Library:

```bash
# Find your .NET runtime path
dotnet --list-runtimes
# Microsoft.NETCore.App 10.0.0 [/usr/share/dotnet/shared/Microsoft.NETCore.App]

# Generate declarations
dotnet run --project src/tsbindgen/tsbindgen.csproj -- \
  generate -d /usr/share/dotnet/shared/Microsoft.NETCore.App/10.0.0 \
  -o ./output
```

### Generate for Custom Assembly

```bash
dotnet run --project src/tsbindgen/tsbindgen.csproj -- \
  generate \
  -a ./path/to/MyLibrary.dll \
  -d /usr/share/dotnet/shared/Microsoft.NETCore.App/10.0.0 \
  -o ./output
```

### Multiple Assemblies

```bash
dotnet run -- generate \
  -a ./Library1.dll \
  -a ./Library2.dll \
  -d $DOTNET_RUNTIME \
  -o ./output
```

## CLI Reference

### The `generate` Command

```
tsbindgen generate [options]
```

### Required Options (at least one)

| Option | Short | Description |
|--------|-------|-------------|
| `--assembly <path>` | `-a` | Path to assembly file to process. Can be specified multiple times. |
| `--assembly-dir <path>` | `-d` | Directory containing .NET runtime assemblies. Required for type resolution. |

### Output Options

| Option | Short | Default | Description |
|--------|-------|---------|-------------|
| `--out-dir <path>` | `-o` | `out` | Output directory for generated files. Created if it doesn't exist. |

### Filtering Options

| Option | Short | Description |
|--------|-------|-------------|
| `--namespaces <list>` | `-n` | Comma-separated list of namespaces to generate. If omitted, generates all namespaces. |

**Example:**
```bash
# Only generate System and System.Collections.Generic
dotnet run -- generate -d $DOTNET_RUNTIME -o ./out \
  -n System,System.Collections.Generic
```

### Naming Options

| Option | Values | Default | Description |
|--------|--------|---------|-------------|
| `--naming` | `js`, `clr` | `clr` | Member naming convention. |

**Naming modes:**

- **`clr`** (default): PascalCase matching C# conventions
  ```typescript
  list.GetEnumerator();
  Console.WriteLine("hello");
  ```

- **`js`**: camelCase matching JavaScript conventions
  ```typescript
  list.getEnumerator();
  console.writeLine("hello");
  ```

### Library Mode Options

| Option | Description |
|--------|-------------|
| `--lib <path>` | Path to pre-existing BCL type declarations. Enables library mode. |

See [Library Mode](#library-mode) for details.

### Diagnostic Options

| Option | Short | Description |
|--------|-------|-------------|
| `--verbose` | `-v` | Enable detailed progress output. |
| `--logs <categories>` | - | Enable specific log categories (comma-separated). |
| `--strict` | - | Enable strict mode validation with additional checks. |

**Available log categories:**
- `ImportPlanner` - Import statement planning
- `FacadeEmitter` - Facade file generation
- `InternalIndexEmitter` - Internal declaration generation
- `ViewPlanner` - Explicit interface view planning
- `StructuralConformance` - Interface conformance analysis
- `SafeToExtendAnalyzer` - LINQ assignability analysis

**Example:**
```bash
dotnet run -- generate -d $DOTNET_RUNTIME -o ./out \
  -v --logs ImportPlanner,FacadeEmitter
```

## Output Structure

### Directory Layout

```
output/
├── System/
│   ├── index.d.ts                    # Public facade
│   └── internal/
│       └── index.d.ts                # Type declarations
├── System.Collections.Generic/
│   ├── index.d.ts
│   └── internal/
│       └── index.d.ts
├── System.IO/
│   ├── index.d.ts
│   └── internal/
│       └── index.d.ts
└── ... (one directory per namespace)
```

### File Types

#### `index.d.ts` (Facade)

The public-facing module that consumers import from:

```typescript
// Generated by tsbindgen - Architecture
// Namespace: System.Collections.Generic

import * as Internal from './internal/index.js';

// Cross-namespace type imports
import type { IEnumerable_1 } from '../System.Collections/index.js';

// Re-export everything from internal
export * from './internal/index.js';

// Individual type exports with friendly aliases
export type List<T> = Internal.List_1<T>;  // Friendly alias
export { List_1 } from './internal/index.js';
```

#### `internal/index.d.ts` (Declarations)

Full type declarations with the `$instance` pattern:

```typescript
// Generated by tsbindgen - Architecture
// Namespace: System.Collections.Generic

import type { int, long } from '@tsonic/types';
import * as System_Internal from '../../System/internal/index.js';

// Type declarations
export interface List_1$instance<T> {
  readonly count: int;
  add(item: T): void;
  clear(): void;
  contains(item: T): boolean;
  // ...
}

// Value export (enables new List<T>())
export declare const List_1: {
  new <T>(): List_1<T>;
  new <T>(capacity: int): List_1<T>;
};

// Type alias
export type List_1<T> = List_1$instance<T> & __List_1$views<T>;
```

## Type Mappings

### Primitive Types

| CLR Type | TypeScript | Brand |
|----------|-----------|-------|
| `System.SByte` | `sbyte` | `number & { __brand: "sbyte" }` |
| `System.Byte` | `byte` | `number & { __brand: "byte" }` |
| `System.Int16` | `short` | `number & { __brand: "short" }` |
| `System.UInt16` | `ushort` | `number & { __brand: "ushort" }` |
| `System.Int32` | `int` | `number & { __brand: "int" }` |
| `System.UInt32` | `uint` | `number & { __brand: "uint" }` |
| `System.Int64` | `long` | `bigint & { __brand: "long" }` |
| `System.UInt64` | `ulong` | `bigint & { __brand: "ulong" }` |
| `System.Single` | `float` | `number & { __brand: "float" }` |
| `System.Double` | `double` | `number & { __brand: "double" }` |
| `System.Decimal` | `decimal` | `number & { __brand: "decimal" }` |
| `System.Char` | `char` | `string & { __brand: "char" }` |
| `System.Boolean` | `boolean` | (native) |
| `System.String` | `string` | (native) |
| `System.IntPtr` | `nint` | `number & { __brand: "nint" }` |
| `System.UIntPtr` | `nuint` | `number & { __brand: "nuint" }` |

### Generic Types

Generic type names include arity suffix:

| CLR | TypeScript |
|-----|-----------|
| `List<T>` | `List_1<T>` |
| `Dictionary<K,V>` | `Dictionary_2<K, V>` |
| `Tuple<T1,T2,T3>` | `Tuple_3<T1, T2, T3>` |
| `Action` | `Action` |
| `Action<T>` | `Action_1<T>` |
| `Func<TResult>` | `Func_1<TResult>` |
| `Func<T,TResult>` | `Func_2<T, TResult>` |

**Friendly aliases** are also exported:
```typescript
import { List } from "@tsonic/dotnet/System.Collections.Generic";  // = List_1
import { Dictionary } from "@tsonic/dotnet/System.Collections.Generic";  // = Dictionary_2
```

### Type Kinds

| CLR Kind | TypeScript Pattern | Example |
|----------|-------------------|---------|
| **Class** | `interface + const` | `interface Exception$instance { ... }` + `const Exception: { new(): Exception }` |
| **Struct** | `interface + const` | `interface DateTime$instance { ... }` + `const DateTime: { ... }` |
| **Interface** | `interface` | `interface IEnumerable_1$instance<T> { ... }` |
| **Enum** | `const enum` | `const enum ConsoleColor { Black = 0, ... }` |
| **Delegate** | `type` (callable) | `type Action_1<T> = ((arg: T) => void) & Action_1$instance<T>` |
| **Static class** | `abstract class` | `abstract class Console { static writeLine(...): void }` |

### Nullable Types

| CLR | TypeScript |
|-----|-----------|
| `int?` | `int \| null` |
| `string?` | `string \| null` |
| `T?` where T: struct | `T \| null` |

### Arrays and Collections

| CLR | TypeScript |
|-----|-----------|
| `T[]` | `T[]` |
| `T[,]` | `T[][]` |
| `IEnumerable<T>` | `IEnumerable_1<T>` |
| `IList<T>` | `IList_1<T>` |
| `ReadOnlySpan<T>` | `ReadOnlySpan_1<T>` |

### Special Types

| CLR | TypeScript |
|-----|-----------|
| `void` | `void` |
| `object` | `unknown` |
| `dynamic` | `unknown` |
| `ref T` | `{ value: ref<T> }` |
| `out T` | `{ value: ref<T> }` |
| `T*` (pointer) | `ptr<T>` |

## Naming Conventions

### CLR Mode (Default)

Members retain their C#/CLR names:

```typescript
// C#: public void WriteLine(string value)
// TS: writeLine(value: string): void  // With --naming js
// TS: WriteLine(value: string): void   // Default (CLR)

Console.WriteLine("Hello");
list.GetEnumerator();
string.IsNullOrEmpty(s);
```

### JavaScript Mode (`--naming js`)

Members are converted to camelCase:

```typescript
console.writeLine("Hello");
list.getEnumerator();
string.isNullOrEmpty(s);
```

**Conversion rules:**
- First letter lowercase
- Acronyms preserved: `XMLReader` → `xmlReader`
- Single-letter words: `IDisposable` → `iDisposable`

### Type Names

Type names are **always** preserved in their original form:
- `List_1` (not `list_1`)
- `Dictionary_2` (not `dictionary_2`)
- `IEnumerable_1` (not `ienumerable_1`)

## Library Mode

Library mode generates declarations for your assembly without duplicating BCL types. Instead, it imports BCL types from a pre-existing package.

### Use Case

When building a NuGet package with TypeScript bindings:
1. BCL types come from `@tsonic/dotnet` (published package)
2. Your types are generated fresh with references to BCL

### Workflow

```bash
# Step 1: Generate BCL (or use pre-existing @tsonic/dotnet)
dotnet run -- generate -d $DOTNET_RUNTIME -o ./bcl-types

# Step 2: Generate your library with --lib
dotnet run -- generate \
  -a ./MyLibrary.dll \
  -d $DOTNET_RUNTIME \
  -o ./my-lib-types \
  --lib ./bcl-types
```

### What --lib Does

1. **Skips BCL types** - Types from assemblies in `--lib` are not regenerated
2. **Generates imports** - Cross-references become import statements
3. **Validates references** - Ensures all referenced types exist in `--lib`

### Validation Errors

Library mode performs strict validation:

| Error | Description |
|-------|-------------|
| `LIB001` | Type not found in library package |
| `LIB002` | Member signature mismatch |
| `LIB003` | Generic constraint mismatch |

```bash
# Run full BCL validation

# Expected output:
```


### Regression Tests

```bash
# Run all tests
bash test/scripts/run-all.sh

# Individual tests
bash test/scripts/test-determinism.sh       # Reproducible output
bash test/scripts/test-strict-mode.sh       # Strict mode compliance
bash test/scripts/test-surface-manifest.sh  # API surface stability
bash test/scripts/test-lib.sh               # Library mode
bash test/scripts/test-naming-js.sh         # JavaScript naming
```

### Strict Mode

Enable additional validation checks:

```bash
dotnet run -- generate -d $DOTNET_RUNTIME -o ./out --strict
```

Strict mode validates:
- All members have explicit EmitScope
- No unresolved type references
- No duplicate member names
- All generic constraints are satisfiable

## Troubleshooting

### Common Issues

#### "Assembly not found"

```
Error: Could not load assembly 'System.Private.CoreLib'
```

**Solution:** Ensure `--assembly-dir` points to a valid .NET runtime:
```bash
dotnet --list-runtimes
# Use the path shown, e.g., /usr/share/dotnet/shared/Microsoft.NETCore.App/10.0.0
```

#### "Type not resolved"

```
Warning: Could not resolve type 'MyNamespace.MyType'
```

**Solution:** Include all required assemblies:
```bash
dotnet run -- generate \
  -a ./MyLibrary.dll \
  -a ./MyLibrary.Dependencies.dll \
  -d $DOTNET_RUNTIME \
  -o ./out
```

#### "Namespace filter empty"

```
Warning: No types found for namespace filter: MyNamespace
```

**Solution:** Check namespace spelling and ensure assembly contains the namespace:
```bash
# List namespaces in assembly
dotnet run -- generate -a ./MyLibrary.dll -d $DOTNET_RUNTIME -o ./out -v
```

### Performance Tips

1. **Use release build** for large assemblies:
   ```bash
   dotnet build -c Release
   dotnet run -c Release -- generate ...
   ```

2. **Filter namespaces** when only specific types are needed:
   ```bash
   dotnet run -- generate -d $DOTNET_RUNTIME -o ./out \
     -n System.Collections.Generic,System.Linq
   ```

3. **Parallel builds** - Output is deterministic and can be cached
